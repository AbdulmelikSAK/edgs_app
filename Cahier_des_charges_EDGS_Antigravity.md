# Cahier des charges complet - Projet EDGS

## 1. Vision

Cette section définit les exigences de **Vision**.

### Exigences fonctionnelles

-   Solution destinée à EDGS (entreprise de sablage).
-   Une tablette Android par camion (2 à 3 employés maximum).
-   Authentification simple par code PIN.
-   Les missions sont créées exclusivement depuis le back-office.
-   Début de journée, fin de journée et fin de mission avec pointage.
-   Géolocalisation au pointage et toutes les 30 minutes.
-   Carte temps réel des camions lorsqu'une connexion est disponible.
-   Historique GPS complet.
-   Fonctionnement hors ligne avec synchronisation automatique.
-   Photos pendant et après chantier.
-   Gestion des sacs de sable et alertes de stock.
-   Génération d'un rapport PDF.
-   Statistiques détaillées (temps, distance, rentabilité, surface,
    consommation).
-   Interface pensée pour être utilisée avec des gants et en extérieur.

### Exigences techniques

Antigravity doit choisir la stack la plus adaptée (React Native ou
Flutter, backend Laravel, NestJS ou équivalent), puis rester cohérent
sur l'ensemble du projet. Le code doit être modulaire, documenté,
testable et évolutif. Utiliser PostgreSQL ou équivalent, stockage objet
économique compatible S3 pour les photos, architecture API REST
sécurisée, génération d'APK release.

### Critères d'acceptation

Toutes les fonctionnalités doivent être démontrables, documentées et
accompagnées des commandes d'installation, de build et de déploiement.
